# 大宗商品数据 · 自动更新部署说明

## 第一步：上传文件到 GitHub

把以下所有文件上传到你的仓库 `jixingwf/copper-tracker`：

```
scraper.py
fetch_cftc.py
fetch_cme_comex.py
import_lme.py
import_nonfer.py
import_history.py
.github/
  workflows/
    daily_scraper.yml     ← 这个文件决定自动运行时间
```

上传方法（GitHub 网页操作）：
1. 打开 https://github.com/jixingwf/copper-tracker
2. 点击 "Add file" → "Upload files"
3. 把所有 .py 文件拖进去，上传
4. 再单独上传 `.github/workflows/daily_scraper.yml`
   （先在网页上手动创建目录：点 "Create new file"，文件名填 `.github/workflows/daily_scraper.yml`，粘贴内容）

---

## 第二步：配置 Secrets（密钥）

1. 打开你的仓库页面
2. 点击顶部 **Settings** 标签
3. 左侧菜单找到 **Secrets and variables** → **Actions**
4. 点击 **"New repository secret"**，依次添加：

| Secret 名称 | 值 |
|------------|-----|
| `SUPABASE_URL` | `https://opgqjxkaocggconjxgpi.supabase.co` |
| `SUPABASE_KEY` | 你的 Supabase **service_role** key |

> 在哪里找 service_role key？
> Supabase 控制台 → 你的项目 → Settings → API → "service_role" 那行 → 点眼睛图标复制

---

## 第三步：手动触发测试

1. 打开仓库的 **Actions** 标签
2. 左侧点 **"大宗商品数据 · 每日自动抓取"**
3. 右侧点 **"Run workflow"** → **"Run workflow"**
4. 等约 2 分钟，看是否全部绿色 ✅

## 常见问题

**Q: Actions 页面显示红色 ❌ 怎么办？**
点进去查看日志，最常见原因：
- Secrets 没配置或填错
- 某个数据源网站临时下线（`continue-on-error: true` 会跳过不影响其他）

**Q: 想立即更新数据？**
Actions → Run workflow 手动触发，选 "all" 运行全部任务。

**Q: 有色金属数据多久更新一次？**
每周五更新，因为 SHFE 库存是周五发布，LME 持仓也是周度数据。
